# Introduction

Hello, today you will be testing the gui of our artifact mining application. Please provide any feedback you can think of during your time using the app and there will be a form at the end of the process for you to fill out. We appreciate any feedback you have to offer and encourage you to be honest with your answers.

# Tasks

**Task 1:** Navigate through the differente tabs and familiarize yourself with the application.

**Task 2:** Scan the folder located at `"tests\testdata\test_peer_testing\testScanFolder.zip"`

**Task 3:** Find and explore the projects in the log generated from your scan.

**Task 4:** Apply a thumbnail to a project in the log

**Task 5:** Add or remove a file from a project in the log

**Task 6:** Create a new project and add a file to that project

**Task 7:** Navigate to the Resume/Portfolio page and generate a resume and portfolio

**Task 8:** Navigate to the items page and view the resume and portfolio you generated

**Task 9:** Modify a project in the Resume/Portfolio page, save changes, and regenerate a resume or portfolio

**Task 10:** View the modified resume or portfolio

**Task 11:** Complete the google form questionnaire

# Things to note during the test

- Try focusing on which parts testers get stuck on, and where they seem to struggle the most.
- The amount of time it takes the tester to learn a new component of the system.
- Which tasks the user struggles with the most.
- Any errors or bugs they encounter when using the app.
- Any other comments or feedback given by the tester during the test.

# Notes

Accept any feedback given from the tester. All information is useful and should not be neglected or overlooked if we disagree with the tester. Peer test is meant to collect information on how we can improve our application by learning about the user experience with our app.
